INTRODUCCIÓN

Una clase es un tipo de dato que contiene código (funciones) y datos. Una clase permite encapsular todo el código y los datos necesarios para gestionar un tipo específico de un elemento de programa.

Para hablar de clases, primero debemos hablar de "objetos", ya que de éstos se componen las clases. La programación orientada a objetos nació en 1969 por el noruego Kristin Nygaard que intentó escribír programas con varios elementos. Descrubrió que este tipo de problemas era más fácil de manejar si relacionaba cada uno de los elementos con sus acciones entre sí. De ésta manera creó una combinación de la descripción de los elementos en un entorno de proceso de datos.

¿QUE SON LOS OBJETOS?

Un objeto es un elemento con diferentes características, las cuales son almacenadas como datos. Un objeto también ejecuta acciones con sus propios datos. Un programa orientado a objetos puede manejar diferentes objetos. Por ejemplo, un programa que maneja el inventario de una fábrica, la cual cada elemento de la fabrica la reconoce como un objeto, con diferentes caracteristicas y acciones. El prográma manipularía los datos de cada objeto para cumplir con una función determinada.

Los objetos se almacenan en la memoria principal del computador y el programa los llama mientras se está ejecutando. Cuando el programa termina de ejecutar, los objetos vuelven a la memoria.

¿QUE SON LAS CLASES?

Una clase se puede define como: un conjunto de objetos que comparten una esctructura y comportamiento en común.

Antes de crear objetos de cualquier clase, ésta debe ser definida. Se debe nombrar la clase, nombrar los elementos que almacenan sus datos y describír las funciones consideradas en los objetos. El nombramiento de la clase se utiliza para asignar almacenamiento a los datos del programa y reconocer las funciones qeu utilizará el programa. Se define en archivos con un nombre de archivo ".h".

Una declaración de una clase consta de una palabra "class" y el nombre de la clase, seguido de funciones y datos que definen la clase como se muestra en la imagen 1.

MIEMBROS PÚBLICOS Y PRIVADOS

Un principio fundamental en programación orientada a objetos es la ocultación de la información que significa que determinados datos del interior de una clase no se puede acceder por funciones externas a la clase. El mecanismo principal para ocultar datos es ponerlos en una clase y haccerlos privados. A los datos o funciones privados sólo se puede acceder desde dentro e la clase. Por el contrario, los datos o funciones públicas son accesibles desde el exterior de la clase.
